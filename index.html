GitHub Pages 404 Hatası Çözümü

"There isn't a GitHub Pages site here" hatasını düzeltme rehberi

1

GitHub Repository Ayarlarını Kontrol Edin

GitHub'da repository'nize gidin ve "Settings" sekmesine tıklayın.

GitHub Repository Settings Sayfası Görüntüsü
2

Pages Sekmesine Gidin

Sol menüden "Pages" seçeneğini bulun ve tıklayın.

GitHub Pages Ayarları Görüntüsü
3

Doğru Branch'ı Seçin

"Branch" bölümünden "main" veya "gh-pages" branch'ını seçin ve "/ (root)" klasörünü seçtiğinizden emin olun.

Çoğu durumda "main" branch'ını ve "/(root)" klasörünü seçmeniz yeterlidir.
Branch Seçim Ekranı Görüntüsü
4

Kaydet ve Bekle

"Save" butonuna tıklayın ve GitHub Pages'in etkinleşmesini bekleyin. Bu işlem birkaç dakika sürebilir.

GitHub Pages'in tamamen etkinleşmesi 1-10 dakika arasında sürebilir. Lütfen sabırla bekleyin.
5

Repository Adını Kontrol Edin

Kullanıcı sayfası için repository adının kullaniciadi.github.io formatında olması gerekir.

kullaniciadi.github.io
Proje sayfası için ise herhangi bir isim kullanabilirsiniz, ancak erişim adresi kullaniciadi.github.io/repository-adi şeklinde olacaktır.
6

index.html Dosyasını Kontrol Edin

Repository'nizin kök dizininde (ana dizin) index.html dosyası bulunduğundan emin olun.

/index.html
GitHub Pages Dokümantasyonu GitHub'a Git
